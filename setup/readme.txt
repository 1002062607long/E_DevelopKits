
1、本目录中的所有文件列表及说明：

    uninstall.tpl : 被加入到用户程序安装包中，用作生成卸载文件 uninstall.exe 。
    setup.e       : setup.exe 的源程序。
    setup.exe     : 选择非静态编译时，被加入到用户程序安装包中，用做进行具体的安装工作。
    setup_jt.exe  : 选择静态编译时，被加入到用户程序安装包中，用做进行具体的安装工作。
    setup.dll     : 被加入到用户程序安装包中，用作为 setup.exe 提供菜单登记和桌面快捷方式功能。
		    附：由于本文件被 setup.exe 实际使用，所以在执行安装工具生成易语言程序的安
		    装软件时在所需基本文件的列表框中不要取消此文件的选择。
    setup.tpl     : 被加入到用户程序安装包中，用作启动安装程序 setup.exe 。
    mksetup.e     : mksetup.exe 的源程序。
    mksetup.exe   : 未加入到用户程序安装包中，用作组织和生成用户安装包。
    mksetup.dll   : 未加入到用户程序安装包中，用作为 mksetup.exe 提供辅助功能。
    readme.txt    : 本文件

2、如何改进易语言安装工具：
   您可以直接更改所提供的源程序 mksetup.e 和 setup.e ，前一个是用作制作安装包，
后一个用作解析安装包，修改时要注意两者相互对应。比如说如果修改 mksetup.e 在安
装包中增加了某项数据，则就必须修改 setup.e 以解读并处理该项数据。
   修改完后请将被更改的源程序编译成 exe 文件并覆盖到易语言系统的 setup 目录中，
同时要求将最新的 setup.e 文件也复制进去，以便系统分析启动 setup.exe 所需要的
附属文件，否则有可能会导致因缺少支持文件而无法启动 setup.exe 的情况发生。

3、关于窗口xp化：
   如果您在生成安装程序时选择了窗口xp化，则生成的安装程序将附带xplib.fne文件，如果用户程序中已经使用了xplib.fne，则安装程序将只附带一个xplib.fne。如果用户程序和制作安装程序时都不使用xp风格，xplib.fne将不会附带。
